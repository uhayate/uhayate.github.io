面向对象的方法

2018年7月8日

Python在面向对象中提供了三种方法：实例方法、静态方法、类方法

实例方法

实例方法必须绑定到实例对象才可调用。也就是说，实例方法只能由实例对象进行调用。 第一个参数必须要默认传实例对象，一般用self

class Bar(object):
    x = 10

    def instance_method(self):
        print self.x

# 通过实例调用：正确
test = Bar()
test.instance_method()

# 通过类调用：错误
Bar.instance_method()

运行结果

10
Traceback (most recent call last):
  略去...
    Bar.instance_method()
TypeError: unbound method instance_method() must be called with Bar instance as first argument (got nothing instead)

静态方法

概述

静态方法：由@staticmethod装饰的方法 不存在任何与类、实例相关的参数，包括cls、self

class Bar(object):
    x = 10

    @staticmethod
    def instance_method(data):
        print 'instance_method', data

b = Bar()
b.instance_method('xxx')
Bar.instance_method('yyy')

运行结果：

instance_method xxx
instance_method yyy

使用场景

若类的某个函数运行时不会涉及到与类和实例有关的操作时（如获取类、实例属性），即可考虑定义为静态方法

类方法

概述

类方法：由@classmethod装饰的方法 第一个参数必须要传类对象，一般用cls

class Bar(object):
    x = 10

    def __init__(self, y):
        self.y = y

    @classmethod
    def class_method(cls):
        print 'class_method', cls.x

b = Bar(20)
b.class_method()

Bar.class_method()

运行结果

class_method 10
class_method 10

使用场景

强调简单地直接用类来调用函数，而不用借助类实例 在运行过程中，不使用到实例的属性。如上代码，class_method()是无法访问类实例化时传入的y属性的

静态方法、类方法

相同点

  • 都可以被类和实例调用

不同点

  • 静态方法：不该访问类属性，无法访问实例的属性 为什说不该访问类属性，看如下代码：
class Bar(object):
    x = 10

    @staticmethod
    def static_method():
        print 'static_method', Bar.x

    @classmethod
    def class_method(cls):
        print 'class_method', cls.x

b = Bar()
b.static_method()
b.class_method()

通过静态方法也可以访问类属性，可以做到跟类方法一样的效果，但是这样的话，在类名被修改时，就需要改动代码。这种情况，就该使用类方法

  • 类方法：无法访问实例的属性，只能访问类的属性
